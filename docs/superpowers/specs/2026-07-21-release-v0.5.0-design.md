# Release v0.5.0 — Design

Date: 2026-07-21
Branch: `20260721-release`
Author: Lanh Hoang

## Context

Ninety-five commits have landed on the `master` branch since `v0.4.0` (2026-07-16).
The work splits into four refactor phases plus a tools-dashboard refactor:

- **Phase 1 — provider surface cleanup**: removed the broken Exa MCP provider and the `openai-native` compatibility alias; renamed the OpenAI web-search rewrite module to `openai-web-search-rewrite`.
- **Phase 2 — OpenAI Codex OAuth**: `openai-codex` now resolves credentials through Pi's active `ModelRegistry`. There is no longer an `OPENAI_API_KEY` fallback for codex; users who want a key-backed search should configure `openai-web-search` instead.
- **Phase 3 — usage storage and config cleanup**: provider usage now persists to `$PI_CODING_AGENT_DIR/cache/pi-tools/usage.json` using a version 2 UTC-period ledger. Legacy `pi-tools.json` filename fallbacks and `monthlyQuota` settings were removed.
- **Phase 4 — provider budgets**: hard, managed, unlimited, and shared-pool budgets replaced the old request-count quotas. Budget status is exposed through `/tools` and the provider registry.
- **Tools dashboard refactor (phases 1–4)**: typed `/tools` subcommands (`status`, `enable`, `disable`, `key`, `test`, `default`, `monitor on/off`) were replaced by a single interactive `/tools` TUI dashboard with Status, Providers, Test, and Activity tabs. The command now warns users who pass arguments: "no longer supports typed subcommands."

The current `README.md` still documents the typed subcommands that no longer exist. `release.yml` still publishes with `NPM_TOKEN`, which is unnecessary now that the workflow already has `id-token: write`. `package.json` is still at `0.4.0`. The `CHANGELOG.md` `[Unreleased]` section is comprehensive but does not mention the dashboard refactor.

This spec describes the prep work to ship `v0.5.0`.

## Goals

- Cut `v0.5.0` with all current Unreleased work captured under one release entry.
- Update `README.md` to reflect the dashboard-first user model and split the example config into a focused minimal config plus a full reference.
- Migrate the release workflow to npm trusted publishing (drop the `NPM_TOKEN` secret requirement).
- Make each release-prep change an independently reviewable commit.

## Non-goals

- Creating the git tag or pushing the branch/tag (the user handles release shipping).
- Configuring the Trusted Publisher on npmjs.com (one-time manual step assumed already complete).
- Restructuring the project beyond these four file changes.
- Adding new features or fixes during the release prep.

## Approach

Three independent commits on the `20260721-release` branch:

1. **`chore(ci): migrate release workflow to npm trusted publishing`** — `.github/workflows/release.yml` only.
2. **`chore(release): prepare v0.5.0`** — `package.json` and `CHANGELOG.md`.
3. **`docs(readme): rewrite around tools dashboard for v0.5.0`** — `README.md` only.

Each commit is independently reviewable and revertable.

## Detailed changes

### Commit 1 — release workflow migration

File: `.github/workflows/release.yml`

Remove the `env:` block from the "Publish to npm" step. The publish step becomes:

```yaml
      - name: Publish to npm
        run: npm publish --provenance --access public
```

Keep:

- `id-token: write` (required for OIDC token exchange).
- `contents: read`.
- The "Verify tag matches package version" guard.
- `--provenance --access public` (provenance is generated automatically for public packages from public repos under trusted publishing; keeping the flag is harmless and matches the prior intent).

Do not add `NODE_AUTH_TOKEN` or any token reference.

### Commit 2 — package and changelog bump

File: `package.json`

- Change `"version": "0.4.0"` to `"0.5.0"`.
- Leave description, keywords, scripts, dependencies, devDependencies, peerDependencies, files, engines, and the `pi` extension entry unchanged.

File: `CHANGELOG.md`

- Insert a fresh `## [Unreleased]` section at the top (currently empty placeholder).
- Rename the existing `## [Unreleased]` heading to `## [0.5.0] - 2026-07-21`.
- Under `### Changed` in the new `0.5.0` entry, add one bullet at the top of the Changed list:
  ```
  - Replaced typed `/tools` subcommands (status, enable, key, test, default, monitor) with a single interactive `/tools` TUI dashboard.
  ```
- Leave the remaining bullets in their existing order. They accurately reflect the 95 commits since 0.4.0.

### Commit 3 — README rewrite

File: `README.md`

Target structure (in order):

1. Title and badges (unchanged).
2. Short intro paragraph (unchanged).
3. **New section — "What's new in 0.5.0"** (short callout):
   - Tools dashboard replaces typed `/tools` subcommands (Providers, Status, Activity tabs).
   - Provider budgets are enforceable.
   - `openai-codex` requires Pi OAuth; the `OPENAI_API_KEY` fallback is gone.
   - Usage persistence moved to `$PI_CODING_AGENT_DIR/cache/pi-tools/usage.json`.
4. **Install** (unchanged).
5. **Use it** — keep the seven-tool table and per-tool usage examples (unchanged). In the "Search the web" subsection, leave the multi-provider, fallback, and fusion copy intact.
6. **Manage providers** (rewritten):
   - Drop the entire `/tools status|enable|disable|key|test|default|monitor on|monitor off` table.
   - Replace with: "Run `/tools` to open the interactive dashboard. The dashboard has three tabs: Providers (toggle, key, default, run tests), Status (budgets, session outcomes, latency), and Activity (recent provider events). Switch between global and project scope; project scope requires trust or an existing project config."
   - Keep the line about provider configuration refreshing every 30 seconds and using `/tools reload` for an immediate refresh.
7. **Available providers** table (unchanged).
8. **Configure files and credentials** — split into two subsections:
   - **Minimal config** — a small starter JSON (~25 lines) showing the most common fields: `defaultProvider`, a single keyed search provider (`brave` or `exa`), the global config path note, and the `providers` shape. No `deepResearch`, `youtube`, `video`, `pdf`, `gemini`, `combine`, `ssrf`, `guidance`, or `github` sections.
   - **Full configuration reference** — move the current full example block here verbatim, with the comment "Complete shape of the merged configuration."
   - Keep the per-credential-type explanation (`apiKey` can be env-var name, literal, or shell command).
   - Keep the trust-gating note.
9. **Provider budgets** section (unchanged).
10. **Advanced settings** list (unchanged).
11. **Optional command-line tools** table (unchanged).
12. **Development** (unchanged).
13. **Changelog** and **License** (unchanged).

Update one sentence inside the providers section: clarify that `openai-codex` requires Pi OAuth via `/login` (no `OPENAI_API_KEY` fallback), and that key-backed search users should configure `openai-web-search`.

The rewrite does not delete information — the full config reference moves to the bottom, and the dashboard replaces the typed-subcommand table rather than supplementing it.

## Verification

After each commit, run the verification listed below before claiming the commit is complete. The README commit cannot break the build (markdown only), but `pnpm check` is run anyway to confirm no incidental regression.

- Commit 1: `git diff -- .github/workflows/release.yml` shows only the `env:` removal on the publish step. `grep -n "NPM_TOKEN\|NODE_AUTH_TOKEN" .github/workflows/release.yml` returns no matches.
- Commit 2: `git diff -- package.json` shows only the `version` line. `git diff -- CHANGELOG.md` shows: a new empty `[Unreleased]` heading, the `[Unreleased]` → `[0.5.0] - 2026-07-21` rename, and one new bullet at the top of the Changed list.
- Commit 3: `pnpm check` passes. Visual diff: the typed-subcommand table is gone, the "What's new in 0.5.0" callout is present, the Minimal config block appears before the Full reference, the providers section describes the three tabs (Providers, Status, Activity) rather than the old typed subcommands, and the providers section reflects OAuth-only `openai-codex`.

After all three commits, run `pnpm release:check` once to confirm the full pre-release gate (lint, typecheck, tests, and `pnpm pack --dry-run`) passes with the bumped version.
